# NodeXpressTS

A modular, scalable, and fully-typed backend server built with **Node.js**, **Express**, and **TypeScript**. The project emphasizes best practices in security, maintainability, and performance, including features like JWT authentication, user profile management, role-based access control (RBAC), centralized logging, and API documentation with Swagger.

## ‚ú® Features

-   **Authentication**: JWT-based authentication with refresh tokens
-   **User Management**: Complete user profile management, including updating profiles and role-based permissions
-   **Security**: Helmet for security headers, rate limiting, and input validation using Zod
-   **API Documentation**: Integrated Swagger for comprehensive API documentation
-   **Error Handling**: Centralized error management and handling with custom error types
-   **Logging**: Centralized logging using Winston
-   **Validation**: Data validation using Zod
-   **Docker Support**: Ready for deployment with Docker and Docker Compose

## üöÄ Getting Started

### Prerequisites

-   **Node.js** (v16+)
-   **MongoDB**
-   **Docker** (optional)

### Installation

1. Clone the repository:

```bash
git clone https://github.com/gabriiellfr/nodexpressts.git
cd NodeXpressTS
```

2. Install dependencies:

```bash
npm install
```

3. Copy the `.env.example` file to create your own `.env` file:

```bash
cp .env.example .env
```

4. Set up the necessary environment variables in `.env` (see [Environment Variables](#-environment-variables)).

## ‚öôÔ∏è Environment Variables

Ensure your `.env` file contains the following environment variables:

```makefile
PORT=3000
MONGO_URI=mongodb://localhost:27017/nodexpress
JWT_SECRET=your_jwt_secret
JWT_EXPIRES_IN=1h
REFRESH_TOKEN_SECRET=your_refresh_token_secret
REFRESH_TOKEN_EXPIRES_IN=7d
```

Modify these values according to your environment.

## üíª Running the Application

### Development Mode

To run the project in development mode with hot-reloading:

```bash
npm run dev
```

The application will run at `http://localhost:3000`.

### Production Mode

To build and run the project in production mode:

```bash
npm run build
npm start
```

## üê≥ Docker Setup

1. Build the Docker image:

```bash
docker-compose build
```

2. Run the container:

```bash
docker-compose up
```

3. The application should be running on `http://localhost:3000`.
